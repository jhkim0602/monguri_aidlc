### 4.1 EP-09 배치
- 공통 인증·계약·상관관계·감사·마스킹 기반은 U01이 담당한다.
- 공통 네트워크·다중 AZ·관측 수집·암호화·비밀·백업·배포·복구 기반은 U10이 담당한다.
- AI/비동기 상태와 의존성 격리는 U07, 영상 삭제 감시는 U08, 사고 기록·운영 재처리는 U05, 반응형 흐름은 U06이 Primary다.
- 기능별 복원력·PBT는 표의 Collaborating Units를 포함한 실제 소유 Unit이 구현한다. U01/U10이 기능 불변식이나 Service 런타임 책임을 대행하지 않는다.

### 4.2 복원력 15개 규칙
RESILIENCY-01~15는 `unit-of-work.md`의 책임표에 따라 전체 Unit과 Functional/NFR/Infrastructure/Code/Build 단계로 전달됐다. 본 스토리 맵에서는 EP-09 12개 스토리가 관측, 비동기 복구, 삭제, 다중 AZ, 백업, 의존성 격리, 롤백, 사고 대응의 사용자 가치를 추적하고 각 기능 스토리의 품질 메모가 권한·멱등·재연결·단계적 저하를 보강한다. **차단 발견 사항: 0.**

### 4.3 부분 PBT
- **PBT-02**: U01~U09의 REST/이벤트/SSE/WebSocket/Port 직렬화 왕복과 U10 구성 변환의 손실 범위를 검증한다.
- **PBT-03**: U01 권한·삭제, U02 근거·버전, U03 동의·객관 리포트, U04 Snapshot·기간, U05 최소 권한, U07 멱등·체크포인트, U08 7일 삭제, U09 Grant·방 격리, U10 복원·구성 불변식을 검증한다.
- **PBT-07**: 각 Unit이 제약을 지키는 도메인 생성기를 소유하고 공통 생성기 기반만 공유한다.
- **PBT-08**: 모든 실패에서 축소된 입력과 seed를 기록하고 CI/Build and Test에서 재현한다.
- **PBT-09**: 언어가 정해지는 Unit별 NFR Requirements에서 프레임워크를 선택하고 버전을 고정한다.
- **차단 발견 사항**: 0. 프레임워크 미선택은 현재 단계의 올바른 후속 전달이며 임의 기술 선택을 하지 않았다.

### 4.4 보안 기준 (Security Baseline)
Security Baseline은 비활성화 상태다. 다만 인증(CAR-01), 소유권·최소 권한(CAR-02/03), 민감 로그 금지(CAR-06), 감사(CAR-07), 데이터 경계·암호화(CAR-08 및 US-09-01), 삭제(CAR-09)는 프로젝트 고유 필수 조건으로 각 Primary/Collaborating Unit에 유지된다.
