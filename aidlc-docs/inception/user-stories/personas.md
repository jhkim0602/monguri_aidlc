# 페르소나

## 1. 페르소나 구성 원칙
승인된 계획에 따라 사용자 유형을 구직 사용자, 전문가, 운영자 세 그룹으로 통합한다. 그룹 안의 행동 차이는 변형 유형으로 표현한다.

## 2. P-01 구직 사용자

### 대표 설명
채용공고를 중심으로 경험, 문서, 면접과 상담을 관리하며 취업 또는 이직을 준비하는 개인 사용자다.

### 변형 유형
- **첫 취업 준비자**: 경험을 직무 언어로 구조화하는 데 어려움이 있고 단계별 안내를 필요로 한다.
- **이직 준비자**: 기존 경력과 프로젝트를 여러 공고에 빠르게 재사용하고 싶어 한다.

### 목표
- 지원할 공고와 마감 일정을 놓치지 않는다.
- 자신의 실제 경험을 한 번 정리해 여러 문서에서 재사용한다.
- 공고에 맞는 이력서와 자기소개서 초안을 빠르게 만든다.
- AI가 어떤 경험을 근거로 문장을 작성했는지 확인한다.
- 실시간 모의면접과 전문가 피드백으로 답변을 개선한다.

### 동기
- 반복적인 문서 정리 시간을 줄이고 싶다.
- 자신의 경험과 무관한 AI 결과를 피하고 싶다.
- 준비 과정과 피드백을 한곳에서 누적하고 싶다.

### 주요 불편
- 공고, 일정, 경험과 문서가 여러 서비스에 흩어져 있다.
- 같은 경험을 공고마다 반복해서 작성한다.
- AI 결과의 사실성과 근거를 판단하기 어렵다.
- 면접 결과를 다음 행동이나 전문가 상담으로 연결하기 어렵다.

### 행동 특성
- 여러 공고를 비교하며 마감일에 맞춰 준비한다.
- 초안을 빠르게 만든 뒤 직접 편집하고 확정한다.
- 민감한 문서와 영상을 공유할 때 대상과 기간을 확인하려 한다.

### 권한과 데이터 경계
- 자신의 공고, 경험, 문서, 면접 및 상담 자료만 기본적으로 접근한다.
- 상담 요청에 포함할 자료를 직접 선택한다.
- AI 결과를 채택, 수정 또는 거절한다.
- 계정과 개인정보 삭제를 요청할 수 있다.

### 관련 Epic
EP-01 계정과 접근, EP-02 공고와 일정, EP-03 경험, EP-04 AI 문서, EP-05 AI 면접, EP-06 전문가 상담
## 3. P-02 전문가

### 대표 설명
사용자가 상담 요청에 첨부한 자료를 검토하고 영상통화, 채팅, 메모와 수정 제안으로 피드백을 제공하는 검증된 전문가다.

### 변형 유형
- **커리어 컨설턴트**: 공고, 경험, 이력서와 자기소개서의 적합성과 표현을 검토한다.
- **면접 코치**: 질문별 영상, 전사문, 답변 내용과 전달 방식을 검토한다.

### 목표
- 상담 전 필요한 자료를 한 화면에서 빠르게 파악한다.
- 허용된 자료만 안전하게 검토한다.
- 플랫폼 내부에서 상담하고 실행 가능한 피드백을 남긴다.
- 예정, 진행 및 완료 상담을 효율적으로 관리한다.

### 동기
- 자료 수집과 상담 준비 시간을 줄이고 싶다.
- 사용자 맥락을 근거로 일관된 피드백을 제공하고 싶다.
- 개인정보와 권한을 명확히 지키며 상담하고 싶다.

### 주요 불편
- 공고, 문서, 영상과 리포트가 서로 다른 채널에 흩어져 있다.
- 어떤 자료가 최신인지 또는 상담용으로 승인됐는지 판단하기 어렵다.
- 상담 중 수정 의견과 후속 조치가 별도로 관리된다.

### 행동 특성
- 상담 전에 요청 목적과 첨부 스냅샷을 검토한다.
- 문서와 리포트를 함께 보며 구체적인 근거를 제시한다.
- 상담 일정과 가능한 시간을 주기적으로 관리한다.

### 권한과 데이터 경계
- 운영자의 신원과 경력 심사를 통과해야 활성화된다.
- 현재 상담 기간에 사용자가 선택한 스냅샷만 열람한다.
- 권한이 만료된 자료와 선택되지 않은 과거 자료에 접근할 수 없다.
- 상담 메모와 수정 제안은 허용된 상담 맥락에만 작성한다.

### 관련 Epic
EP-01 계정과 접근, EP-06 전문가 상담, EP-07 전문가 업무

## 4. P-03 운영자

### 대표 설명
사용자와 전문가 계정, 전문가 검증, 신고, 권한, 콘텐츠 및 상담 운영 상태를 관리하는 플랫폼 담당자다.

### 목표
- 전문가의 신원과 경력을 일관된 기준으로 심사한다.
- 신고와 운영 장애를 추적하고 적절히 처리한다.
- 민감 자료 접근과 자동 삭제 실패를 감사한다.
- 서비스 상태, 백업과 복구 가능성을 확인한다.

### 동기
- 제한된 팀 인력으로 서비스 신뢰성과 개인정보 보호를 유지하고 싶다.
- 장애와 사용자 문제를 재현 가능한 기록으로 남기고 싶다.
- 포트폴리오 규모에 맞는 단순한 운영 절차를 사용하고 싶다.

### 주요 불편
- 권한 만료, 영상 삭제, 이메일 실패 같은 비동기 작업을 놓치기 쉽다.
- 민감정보를 과도하게 열람하지 않고 문제를 조사해야 한다.
- 별도의 조직용 변경·장애 관리 도구가 없다.

### 행동 특성
- 계정과 상담 상태를 검색해 예외 상황을 처리한다.
- 경고와 감사 로그를 근거로 운영 문제를 조사한다.
- Git 이력과 경량 장애 기록을 사용한다.

### 권한과 데이터 경계
- 운영 기능은 일반 사용자 및 전문가 기능과 분리된다.
- 민감 자료 열람은 최소 권한과 업무 목적에 한정된다.
- 계정·권한·신고 상태 변경은 감사 로그에 기록된다.

### 관련 Epic
EP-01 계정과 접근, EP-08 플랫폼 운영, EP-09 품질·복원력 지원
